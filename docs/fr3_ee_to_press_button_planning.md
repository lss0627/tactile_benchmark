# FR3 EE-to-PressButton Planning

This gate connects the safe local differential IK envelope to the existing
`PressButton` task geometry. It does not make FR3 press the button, does not
collect data, does not train a model, and does not create benchmark results.

## Starting Point

The preceding FR3 local differential IK gate established:

- FK is available for the configured FR3 kinematics solver.
- A finite-difference translation Jacobian is available with shape `[3, 7]`.
- 0.25 mm, 0.5 mm, and 1 mm +/-X and +/-Z differential IK target checks are
  bounded.
- One 0.25 mm +X free-space differential IK runtime smoke completed.
- Reports explicitly record `uses_lula_global_ik=false` and
  `uses_joint_space_fallback=false`.

The canonical differential IK report path is:

`outputs/fr3_differential_ik/target_report.json`

`scripts/check_fr3_press_button_approach_readiness.py` accepts that path
directly and falls back to it when an older
`differential_ik_target_report.json` path is missing. The readiness report
records `diffik_report_exists` and `diffik_report_canonical` so this artifact
binding is auditable.

Lula global target IK is not used as the PressButton control default because
previous 1-5 mm target checks produced nonlocal unsafe arm joint deltas. The
planning path uses local damped least-squares differential IK for tiny
translation substeps.

## PressButton Geometry

Geometry is configured in:

`configs/tasks/press_button_fr3_planned.yaml`

The report generated by `scripts/check_press_button_geometry.py` records:

- `button_position=[0.55, 0.0, 0.47]`;
- `button_normal=[0.0, 0.0, 1.0]`;
- `button_press_axis=[0.0, 0.0, -1.0]`;
- `button_press_depth=0.03`;
- `pre_press_offset=0.08`;
- `near_contact_offset=0.012`;
- `retreat_distance=0.12`;
- `success_source=button_displacement`;
- `contact_force_available=false`;
- `force_source=unavailable`;
- `uses_fake_force=false`.

The geometry source is explicitly marked as planned/configured from the current
PressButton runtime scene constants, not as a measured sensor estimate.

## Co-Scene Load-Only

`scripts/run_fr3_press_button_load_only_smoke.py` prepares a load-only
co-scene with `/World/FR3`, a table, and a primitive red button. Dry-run mode
does not start Isaac Sim. Runtime mode may start Isaac Sim and save a
screenshot, but it still does not send joint commands or move the EE.

The status records:

- `fr3_loaded`;
- `press_button_loaded`;
- `ee_frame`;
- `button_frame`;
- `ee_to_button_vector`;
- `ee_to_button_distance`;
- `joint_command_sent=false`;
- `ee_motion_executed=false`;
- `button_pressed=false`;
- `benchmark_result=false`.

## Waypoint Plan

`scripts/plan_fr3_press_button_waypoints.py` creates the no-command waypoint
sequence:

- `home`;
- `pre_press`;
- `near_contact`;
- `press_target`;
- `hold_pressed`;
- `retract`;
- `return_safe`.

The planner approaches along the reverse press axis, uses the configured press
depth, splits each segment into tiny Cartesian substeps, and defaults to the
previously validated 0.25 mm step size. Dry-run checks geometry/workspace
bounds only. Runtime no-command planning can read the FR3 Jacobian and evaluate
damped least-squares `dq` for each substep, but it still sends no joint
commands.

## Safety Envelope

The plan must keep:

- `uses_differential_ik=true`;
- `uses_lula_global_ik=false`;
- `uses_joint_space_fallback=false`;
- `joint_command_sent=false`;
- `button_pressed=false`;
- `dataset_collection_allowed=false`.

Force and wrench are not available in this stage. Button displacement remains a
future success hook, not a force signal.

## Approach-Only Runtime Boundary

The next runtime script is:

`scripts/run_fr3_press_button_approach_only_smoke.py`

Allowed modes are `micro_approach`, `short_approach`, `pre_press`, and
`near_contact`. These modes may use the existing differential IK controller to
move toward pre-contact waypoints, but they must not execute `press_target`,
must not execute press depth, must not require button success, and must not
collect a dataset. Output artifacts must record `approach_only=true`,
`press_target_executed=false`, `button_pressed=false`,
`dataset_collection_allowed=false`, `uses_lula_global_ik=false`, and
`uses_joint_space_fallback=false`.

## Current Non-Claims

- FR3 has not pressed the button.
- This is not a PressButton runtime rollout.
- This is not a dataset collection gate.
- This is not a force-aware tactile benchmark.
- This does not connect tactile mounts.
- This does not train or evaluate a policy.
- This is not a paper benchmark result.

## Next Stage

If the planning gate passes, the next stage is **FR3 PressButton Approach-Only
Runtime Smoke**. That stage should move only to `pre_press` or `near_contact`,
not execute the press-depth segment, not require success, not collect a dataset,
and keep all outputs marked `benchmark_result=false` and
`not_for_paper_claims=true`.
