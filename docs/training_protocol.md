# Training Protocol

This document covers two deliberately separate paths:

- dry-run training protocol for all BC skeleton policies;
- the minimal real StateBC training slice on the mock HDF5 dataset.

The StateBC slice runs real supervised MSE optimization and saves model weights.
The dataset is still generated by the mock/stub runtime, so the resulting policy
is not evidence for paper performance claims.

## Scope

Implemented now:

- `configs/train/bc_mock.yaml`
- `configs/train/state_bc_minimal.yaml`
- `scripts/train_bc.py`
- `isaac_tactile_libero.training.config`
- `isaac_tactile_libero.training.logger`
- `isaac_tactile_libero.training.checkpoint`
- `isaac_tactile_libero.training.bc_trainer`
- `isaac_tactile_libero.training.models.StateBCMLP`

PyTorch is optional and is declared only in the `train` extra:
`python -m pip install -e '.[train]'`. Base installation must not require
torch.

`dry_run: false` is currently supported only for `policy_name: state_bc`. Other
BC skeletons fail explicitly rather than pretending to train.

## Dry-Run Flow

`BCTrainer` performs these checks:

- open the HDF5 dataset through `HDF5DatasetReader`
- select the requested baseline policy spec
- build a filtered mock batch through `batch_builder`
- verify 7D action shape
- verify observation filter checks
- verify tactile masks match the policy modality contract
- verify policy allowed modalities

It does not create an optimizer, run backpropagation, update parameters, or save
real weights.

## Minimal StateBC Real Training

`BCTrainer` supports `dry_run: false` for `state_bc` only. It:

- opens the HDF5 dataset through `HDF5DatasetReader`;
- filters observations with the StateBC modality contract;
- extracts only robot-state features: joint position, joint velocity, ee pose,
  and gripper state;
- verifies no vision, tactile, or oracle fields are used;
- trains `StateBCMLP` with MSE loss and Adam;
- writes `checkpoint.pt` with model weights;
- writes `checkpoint.json` with metadata.

The state feature extractor is named
`schema_compatible_mock_robot_state` because the current dataset is still mock
runtime data. It is not a real robot-state estimator.

## Artifacts

`scripts/train_bc.py` writes:

- `train_log.jsonl`
- `train_summary.json`
- `checkpoint_mock.json` for dry-run, or `checkpoint.pt` plus
  `checkpoint.json` for real StateBC training.

Every artifact records:

- dry-run: `dry_run: true`, `is_trained: false`, `mock_or_stub: true`;
- real StateBC on mock data: `dry_run: false`, `is_trained: true`,
  `mock_or_stub: false`, `runtime_env: mock_dataset`,
  `dataset_is_mock: true`, and `not_for_paper_claims: true`.

`train_log.jsonl` contains one JSON object per dry-run step with `epoch`,
`step`, `mock_loss`, `batch_size`, `policy_name`, and `dry_run`.

`train_summary.json` records total steps, epoch count, dataset episode count,
policy name, status, and output paths.

`checkpoint_mock.json` is metadata only. It records the policy spec, dataset
path, dataset schema version, tactile config snapshot hash/summary, action
schema version, observation filter summary, training config, dry-run status,
mock/trained flags, and git commit when available.

`checkpoint.json` for StateBC records the same schema-critical fields plus
model config, `weights_path: checkpoint.pt`, final loss, runtime environment,
dataset mock status, and the explicit paper-claim prohibition.

## Evaluation Loading

`scripts/evaluate.py --checkpoint checkpoint_mock.json` loads dry-run metadata
into the policy skeleton. The evaluated policy remains untrained: `is_trained`
stays `false`, and output metadata records that the loaded checkpoint is
mock/stub.

`scripts/evaluate.py --policy state_bc --checkpoint checkpoint.json` loads
`checkpoint.pt` through the metadata file and evaluates the trained StateBC
policy in the mock runtime. Evaluation output records `is_trained: true`,
`dataset_is_mock: true`, and `not_for_paper_claims: true`.

## Future Real Training

Future VisionBC/VisionForceBC/VisionVTBC/VisionForceVTBC training can reuse this
config, logging, checkpoint, and evaluation loading protocol. ACT, Diffusion,
VLA, OpenVLA, SmolVLA, and LeRobot policies are not implemented here.

When real training is added, it must use a different checkpoint type and must
not overwrite the semantics of `checkpoint_mock.json`.

## Non-Claims

- Mock checkpoints are not trained models.
- StateBC checkpoints trained on the mock dataset are real training artifacts,
  but not real benchmark results.
- Dry-run metrics are not benchmark scores.
- Mock-dataset trained StateBC evaluation is a software sanity check only.
- Dry-run outputs must not be used for paper performance conclusions.
- No Isaac Sim, Lightwheel, large model, or real optimization stack is active in
  the dry-run path.
