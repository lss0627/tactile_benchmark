# ReplayPolicy Mock/Stub Contract

`ReplayPolicy` is a dataset-driven sanity baseline for Isaac-Tactile-LIBERO. It
reads saved 7D actions from an HDF5 episode and emits them step by step through
the same policy interface used by mock random evaluation.

It is not behavior cloning, imitation learning, ACT, Diffusion Policy, VLA, or a
trained model. It does not infer actions, recover missing states, or claim
physical replay fidelity.

## Public API

`ReplayPolicy` lives at `isaac_tactile_libero.policies.replay` and is registered
under policy name `replay`.

```python
from isaac_tactile_libero.policies.replay import ReplayPolicy

policy = ReplayPolicy(cfg={"dataset": "outputs/mock_dataset/mock_v0.hdf5"})
policy.reset(
    task_name="PegInsert",
    tactile_mode="force_wrench",
    seed=0,
    episode_id="mock-PegInsert-force_wrench-seed0-ep0",
)
action = policy.act(obs)
```

`reset(task_name, tactile_mode, seed, episode_id)` loads one dataset episode. If
`episode_id` is omitted, the first episode matching task, tactile mode, and seed
is selected. `act(obs)` returns the next stored action as a schema-validated
float32 vector with shape `(7,)`. Calling `act()` after the episode is exhausted
raises `StopIteration`.

## Dataset-Driven Evaluation

`scripts/evaluate.py` supports both current policies:

```bash
python scripts/evaluate.py \
  --policy replay \
  --dataset outputs/mock_dataset/mock_v0.hdf5 \
  --max-episodes 10 \
  --output outputs/replay_eval
```

Replay evaluation writes the same `metrics.json` and `summary.csv` files as
random mock evaluation. Each replay episode record includes:

- `policy_name`
- `dataset_path`
- `episode_id`
- `task_name`
- `tactile_mode`
- `seed`
- `replay_consistency`

The replay consistency block checks action shape, step count versus episode
length, dataset observation schema, and episode metric presence. This is a mock
runtime consistency check only. The mock environment is allowed not to reproduce
real physical state because no real Isaac Sim physics backend is active.

## Config

`configs/eval/replay_mock.yaml` is the default replay mock configuration. It
uses:

- `policy: replay`
- `dataset: outputs/mock_dataset/mock_v0.hdf5`
- `max_episodes: 10`
- `output_dir: outputs/replay_eval`

## Boundary

Current replay operates on mock/stub HDF5 episodes generated by
`scripts/collect_mock_demos.py`. The resulting metrics must not be used as paper
performance conclusions.

Future Isaac Sim / Lightwheel backends should reuse this policy/evaluate API and
the same HDF5 schema. The data source may change from mock arrays to real
simulator or sensor streams, but the 7D action schema, observation schema,
tactile schema, episode ids, and evaluation output keys should remain stable
unless the benchmark schema version is intentionally bumped.
